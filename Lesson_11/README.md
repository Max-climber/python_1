# Lesson 11 
#test_api 
API‑тесты в проекте были направлены на проверку функциональности и надёжности серверного API КиноПоиска без участия пользовательского интерфейса. 
Цель — убедиться, что API возвращает корректные, актуальные и безопасные данные, а клиентские приложения (включая UI) могут стабильно с ним взаимодействовать.
Ниже — ключевые направления тестирования.

Поиск по названию фильма/сериала:
- передача запроса через параметр;
- валидация структуры ответа (JSON);
- проверка наличия полей: title, year, rating, poster_url и др.

Фильтрация по жанру:
- запрос с параметром genre;
- проверка, что все возвращённые элементы соответствуют указанному жанру.

Фильтрация по году выпуска:
- запрос с year_from и year_to;
- контроль диапазона дат в результатах.

Комбинирование фильтров (жанр + год, жанр + тип контента и т. п.)
- проверка пересечения условий;
- корректность пагинации при больших выборках.

Валидация структуры и содержимого ответов
Соответствие схемы JSON:
- наличие обязательных полей;
- типы данных (строка, число, массив);
- вложенные объекты (например, directors, actors).

Актуальность данных:
- совпадение рейтингов с веб‑версией;
- актуальность постеров (проверка URL на 200 OK);
- отсутствие «битых» ссылок или пустых полей.

Обработка крайних случаев:
- пустой результат (404 или пустой массив);
- некорректный запрос (400 Bad Request);
- превышение лимита запросов (400 Too Many Requests).

Тестирование аутентификации и авторизации
Проверка API‑токена:
- успешная авторизация с валидным токен;
- отказ при отсутствующем/некорректном токен (401 Unauthorized);
- срок действия токена.

Ограничение доступа:
- запрет на запросы без прав (например, к админ‑API);
- проверка ролей (если API различает пользователей).

Проверка пагинации и сортировки:
Пагинация:
- параметры page и limit;
- наличие next_page / prev_page в ответе;
- корректность общего количества элементов (total_count).

Сортировка:
- по рейтингу (sort=rating), году (sort=year), алфавиту (sort=title);
- направление (order=asc / order=desc).

Тестирование производительности и устойчивости
Время отклика:
- замер времени выполнения запросов;
- выявление «медленных» END point.

Нагрузочное тестирование (опционально):
- отправка серии запросов с интервалом 100 мс;
- проверка устойчивости к DDoS‑подобным нагрузкам.

Повторные запросы:
- обработка дубликатов;
- кэширование ответов (если реализовано).

Обработка ошибок и исключительных ситуаций
Некорректные параметры:
- передача строки вместо числа в year;
- несуществующие жанры/типы контента.

Превышение лимитов:
- слишком большой limit в пагинации;
- частые запросы (проверка rate‑limiting).

Сетевые ошибки:
- имитация разрыва соединения;
- таймауты (проверка timeout в HTTP‑клиенте).

Проверка совместимости и версий
Секционирование API:
- тестирование разных версий (/api/v1/, /api/v2/);
- обратная совместимость при обновлении.

Кросс‑клиентская совместимость
- запросы из разных SDK (Python, JS, curl);
- заголовки User-Agent и их влияние.

Безопасность
- Защита от инъекций;
- SQL‑инъекции в параметрах запроса;
- XSS‑векторы в строковых полях.

Валидация входных данных:
- обрезка/экранирование спецсимволов;
- ограничение длины строк.

HTTPS:
- принудительное использование TLS 1.2+;
- проверка сертификатов.

Итог:
API‑тесты покрывают:
- функциональность (поиск, фильтрация, пагинация);
- надёжность (обработка ошибок, таймауты);
- безопасность (аутентификация, защита от атак);
- производительность (время отклика, нагрузка);
- совместимость (версии API, клиенты).

#test_UI 
UI‑тесты в проекте были направлены на проверку ключевых пользовательских сценариев в веб‑интерфейсе КиноПоиска.
Цель — убедиться, что пользователь может без проблем найти контент, перейти в нужный раздел и взаимодействовать с элементами интерфейса.
Ниже — детализация по основным направлениям.

1. Проверка загрузки и корректности отображения страниц
Главная страница:
- загрузка по базовому URL;
- наличие логотипа и основного меню;
- видимость поля поиска.

1.1 Раздел «Фильмы»:
корректный URL после перехода;
отображение списка карточек фильмов.

1.2 Страница результатов поиска:
появление блока с результатами;
видимость пагинации (если есть).

2. Навигация по сайту
Переход из главного меню в раздел «Фильмы», «Сериалы», «Подборки» и др.
Проверка работоспособности бокового меню и хеддера.
Возврат на главную страницу через логотип.

3. Поиск контента
Ввод запроса в поле поиска (например, «Интерстеллар», «1+1»).
Проверка отображения результатов:
- наличие карточек фильмов/сериалов;
- корректность заголовков в результатах;
- отсутствие пустых или битых карточек.
Тестирование автодополнения (если реализовано).

4. Взаимодействие с элементами интерфейса
Клик по карточке фильма → переход на страницу детализации.
Нажатие кнопки «Буду смотреть» / «Оценить» (если не требует авторизации).
Использование фильтров (жанр, год, рейтинг).
Прокрутка страницы и загрузка дополнительных результатов (если есть бесконечный скролл).

5. Валидация контента
Проверка актуальности данных в карточках:
- название фильма;
- год выпуска;
- рейтинг;
- постер.
Соответствие заголовков страниц поисковому запросу.
Отображение сообщений об отсутствии результатов (если поиск не дал совпадений).

6. Кросс‑браузерная совместимость (опционально)
Проверка работы в Chrome, Firefox (если настроены фикстуры).
Адаптивность интерфейса (настройка окна браузера).

7. Обработка ошибок и крайних случаев
Поиск по пустому запросу.
Поиск по некорректному/несуществующему названию.
Проверка сообщения об отсутствии интернет‑соединения (если эмулируется).
Обработка таймаутов при долгой загрузке элементов.

8. Тестирование интерактивных элементов
Работа слайдеров (например, «Популярное сейчас»).
Всплывающие подсказки и меню при наведении.
Кнопка «Показать ещё» для загрузки дополнительных результатов.

Итог: 
UI‑тесты покрывают:
- базовые сценарии (открытие страниц, навигация);
- ключевые функции (поиск, фильтрация);
- валидацию данных (корректность отображения информации);
- устойчивость к ошибкам (обработка крайних случаев).
