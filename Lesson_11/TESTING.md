# Запуск тестов — тестирование архитектуры сайта

## Для другого пользователя (клонирование с GitHub)

Чтобы получить проект с тестами и запускать их у себя (или тестировать свой деплой):

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/erupto15/python_1.git
   cd python_1/Lesson_11
   ```
2. Установите зависимости: `pip install -r requirements.txt`
3. **Чтобы тестировать свой URL** (например, сайт на GitHub Pages, Vercel или свой сервер), задайте один раз переменную окружения и запускайте тесты:
   ```bash
   export GUIDE_BASE_URL=https://ваш-сайт.example.com
   pytest tests/guide/ -v
   ```
   Или в одну строку:  
   `GUIDE_BASE_URL=https://ваш-сайт.example.com pytest tests/guide/ -v`

После пуша изменений в GitHub локальную копию можно удалить — всё нужное для тестов уже в репозитории.

---

В проекте настроено тестирование **всей архитектуры**:

1. **API-тесты** (`tests/API/`) — проверка внешнего API (КиноПоиск).
2. **UI-тесты КиноПоиска** (`tests/UI/`) — сценарии по сайту КиноПоиск.
3. **UI-тесты сайта guide** (`tests/guide/`) — каталог маршрутов «Скалолазание» (структура, навигация, диалоги).

## Требования

- Python 3.10+
- Chrome и [ChromeDriver](https://chromedriver.chromium.org/) (в PATH или через WebDriver Manager).
- Установка зависимостей:

```bash
cd Lesson_11
pip install -r requirements.txt
```

## Запуск всех тестов

Из каталога `Lesson_11`:

```bash
pytest tests/ -v
```

## Запуск только тестов сайта guide

```bash
pytest tests/guide/ -v
```

Тесты guide по умолчанию открывают `guide/index.html` по локальному пути (`file://`). Если часть тестов (навигация, диалоги) падает — приложение может не инициализироваться до конца при загрузке через `file://`. В таком случае поднимите локальный HTTP-сервер в каталоге `guide/` и задайте базовый URL:

```bash
GUIDE_BASE_URL=http://localhost:8000 pytest tests/guide/ -v
```

(предварительно поднимите сервер в каталоге `guide/`, например: `python -m http.server 8000`).

## Запуск только API- или только UI-тестов (КиноПоиск)

```bash
pytest tests/API/ -v
pytest tests/UI/  -v
```

## Отчёт Allure

Сбор результатов для Allure:

```bash
pytest tests/ --alluredir=allure-results
allure serve allure-results
```

## Что проверяют тесты guide

- **Структура**: загрузка главной, заголовок, шапка, логотип, навигация (Статистика / Топо), поиск, меню пользователя, страница статистики и счётчики.
- **Навигация**: переключение вкладок Статистика ↔ Топо, наличие списка районов и карты на странице «Топо».
- **Диалоги**: открытие диалога входа по клику «Войти», наличие форм входа, регистрации и администратора.
